<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> web API | Untrusted JSON bodies and route params enter signup, login, create-room, and join-room handlers. |
| web API -> database | Validated application decisions become persisted room, team, item, user, and participant records. |
| web API -> room JWT cookie | Database-backed participant authority becomes a signed browser credential. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-S | Spoofing | `POST /api/rooms` | mitigate | Derive Commissioner identity from server session, not from request body role claims. |
| T-01-02-T | Tampering | create-room payload | mitigate | Validate with shared Zod schemas before Prisma writes and keep room/team/item creation transactional. |
| T-01-02-R | Repudiation | participant claims | mitigate | Persist `Participant.anonId`, role, team relation, and timestamps for joins/rejoins. |
| T-01-02-I | Information Disclosure | auth responses | mitigate | Never return `passwordHash`, `ROOM_JWT_SECRET`, or raw internal errors to clients. |
| T-01-02-D | Denial of Service | room code generation | mitigate | Bound collision retry attempts and return typed failure if code generation cannot find a unique code. |
| T-01-02-E | Elevation of Privilege | join route | mitigate | Reject client-submitted Commissioner role and validate Team Owner claim against unclaimed teams. |
</threat_model>
