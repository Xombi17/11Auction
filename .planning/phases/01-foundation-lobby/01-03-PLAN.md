<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> web API | Create and join forms send untrusted user input. |
| browser -> realtime server | Socket connections present untrusted auth tokens and room membership claims. |
| realtime server -> database | Socket lifecycle changes persisted participant presence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-S | Spoofing | Socket.io handshake | mitigate | Verify room-scoped JWT with `ROOM_JWT_SECRET` before joining a Socket.io room. |
| T-01-03-T | Tampering | Lobby client state | mitigate | Render server-returned lobby snapshots and do not mutate team ownership client-side. |
| T-01-03-R | Repudiation | presence changes | mitigate | Update `Participant.connected` and `lastSeenAt` on connect/disconnect. |
| T-01-03-I | Information Disclosure | lobby payload | mitigate | Return display names, roles, team claim state, and connection status only; do not expose password hashes or secrets. |
| T-01-03-D | Denial of Service | socket joins | mitigate | Reject invalid tokens early and keep broadcasts scoped to one room. |
| T-01-03-E | Elevation of Privilege | Commissioner-only UI | mitigate | Do not expose auction start controls in Phase 1, and rely on Phase 2 server-side role checks for future privileged events. |
</threat_model>
